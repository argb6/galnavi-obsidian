# 安全响应头

> [!info] 概述
> GalNavi 由 Cloudflare 边缘返回若干安全相关 HTTP 响应头。不同 Worker 的响应头策略**不一致**——主站/API 与静态页各有侧重。

## 各 Worker 响应头对比（源码确认）

| Worker | Content-Type | Cache-Control | CSP | CORS |
|---|---|---|---|---|
| `websearch.js`（主站 HTML）| `text/html` | — | ✅ 有 | — |
| `websearch.js`（API）| `application/json;charset=UTF-8` | — | — | ✅ `https://galnavi.top` |
| `detail.js`（详情页）| `text/html; charset=utf-8` | — | ❌ | — |
| `about.js`（关于页）| `text/html; charset=utf-8` | ✅ `public, max-age=3600` | ❌ | — |
| `help.js`（帮助页）| `text/html;charset=UTF-8` | — | ❌ | — |

> 只有 `about.js` 显式设置了 `Cache-Control: public, max-age=3600`（1 小时边缘缓存），其他页面未显式设缓存头（由 Cloudflare 默认策略决定）。

## Cloudflare 注入的响应头（实测）

通过 `curl -I` 探测到的 Cloudflare 自动注入头：

```http
HTTP/1.1 200 OK
Server: cloudflare
CF-RAY: a17e...-LAX
Strict-Transport-Security: max-age=0; includeSubDomains
X-Content-Type-Options: nosniff
Content-Type: text/html; charset=utf-8
alt-svc: h3=":443"; ma=86400
Report-To: {"group":"cf-nel", ...}
Nel: {"report_to":"cf-nel", ...}
```

## 安全相关头解析

### Strict-Transport-Security (HSTS)
```http
Strict-Transport-Security: max-age=0; includeSubDomains
```
- **作用**：强制浏览器用 HTTPS 访问
- ⚠️ **`max-age=0`** — 实际是**禁用 HSTS** 的写法（max-age 为 0 表示立即过期）
- `includeSubDomains` — 子域也适用（但因 max-age=0 无实际效果）
- 推测：项目可能处于域名/证书调整期，故意禁用 HSTS 避免锁定。正常应设 `max-age=31536000`（1 年）
- 这是个**待改进点**

### X-Content-Type-Options
```http
X-Content-Type-Options: nosniff
```
- **作用**：禁止浏览器 MIME 类型嗅探
- 浏览器必须严格按 Content-Type 处理响应
- 防止把 text/html 当作可执行脚本执行
- ✅ 正确设置（Cloudflare 自动注入）

### Content-Type
- 主站/API 明确声明 `charset=utf-8`，防 charset 混淆攻击

### alt-svc (HTTP/3)
```http
alt-svc: h3=":443"; ma=86400
```
- 告知浏览器支持 HTTP/3（QUIC），Cloudflare 自动注入，性能优化

### Report-To / Nel
- Network Error Logging，Cloudflare 注入的错误上报机制

## CORS（API 端点）

所有 `/nav/api/*` 端点设置 CORS（源码确认）：
```http
Access-Control-Allow-Origin: https://galnavi.top
```
- **仅允许 galnavi.top 同源读取** API 响应
- 防止其他站点盗用 API
- featured 端点额外支持 OPTIONS 预检：
  ```http
  Access-Control-Allow-Methods: GET, POST, OPTIONS
  Access-Control-Allow-Headers: Content-Type
  ```
- 与 CSP 的 `connect-src 'self'` 形成**双重约束**（见 [内容安全策略 CSP](内容安全策略 CSP.md)）

## Cache-Control 策略

| 页面 | 缓存策略 | 说明 |
|---|---|---|
| about.js | `public, max-age=3600` | 1 小时边缘缓存（内容固定）|
| 其他页面 | 未显式设置 | 依赖 Cloudflare 默认 |
| API | 未显式设置 | 客户端 fetch 用 `cache: 'no-cache'` 强制验证 |

> about.js 设缓存是合理的：关于页内容固定、无动态数据，缓存可减少 Worker 调用。详情页（detail.js）因含网盘密码等可能变更的信息，未设缓存以免过期。

## 未设置的安全头

| 头 | 作用 | 缺失影响 |
|---|---|---|
| X-Frame-Options | 防点击劫持（iframe 嵌入）| 站点可被 iframe |
| Referrer-Policy | 控制 referer 发送 | 默认行为，外链已用 noreferrer |
| Permissions-Policy | 限制浏览器特性 | 未限制摄像头/麦克风等 |

## Cloudflare 层的安全

除响应头外，Cloudflare 自身提供：
- **DDoS 防护**（边缘吸收）
- **challenge-platform**：每个页面末尾注入脚本（`/cdn-cgi/challenge-platform/scripts/jsd/main.js`），做浏览器可信检测
- **TLS 终止**：边缘处理 HTTPS

## 改进建议

1. **HSTS**：将 `max-age=0` 改为 `max-age=31536000; includeSubDomains; preload`（域名稳定后）
2. **CSP 全覆盖**：detail/about/help 页也加 CSP meta
3. **X-Frame-Options** 或 CSP `frame-ancestors 'none'`：防点击劫持
4. **Referrer-Policy: strict-origin-when-cross-origin**：细化 referer 控制
5. **详情页缓存**：detail.js 可考虑短时缓存（如 `max-age=60`）平衡性能与新鲜度
