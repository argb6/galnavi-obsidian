## 二、D1 数据库 — 站点核心数据

```mermaid
erDiagram
    SITES {
        TEXT item_key PK "唯一键 (如 joiplay/灵梦御所)"
        TEXT title "标题"
        TEXT category "分类: simulators/websites/tools/company/hanhua"
        TEXT tags "逗号分隔标签 (如 安卓,rpg,多功能)"
        TEXT short_desc "一句话简介"
        TEXT url "主站链接 (模拟器为空)"
        TEXT icon_path "图标 URL"
        TEXT md_content "详情页 Markdown (仅详情页查询)"
    }

    SITES ||--o{ CATEGORY : belongs_to
    SITES ||--o{ TAGS : has_many

    CATEGORY {
        TEXT name "分类名"
        TEXT frontend "前端映射 (单数)"
        INT count "条目数"
    }

    TAGS {
        TEXT name "标签名"
        TEXT group "标签分组"
        INT frequency "出现频次"
    }
```

## 三、KV 存储 — 配置型数据

```mermaid
graph LR
    subgraph KV Namespace 1 · 轮播图
        HERO_KEY["key: 轮播配置"]
        HERO_VAL["value: JSON 数组<br/>[图片URL1, URL2, URL3]"]
    end

    subgraph KV Namespace 2 · 站长推荐
        FEAT_KEY["key: 推荐配置"]
        FEAT_VAL["value: JSON 数组<br/>[灵梦御所, touchgal, ...]"]
    end

    subgraph 容错机制
        PARSE["JSON.parse"]
        SPLIT["逗号分割"]
        SINGLE["单元素数组"]
        FALLBACK["标签匹配算法<br/>取含'推荐'标签条目"]
    end

    HERO_VAL --> PARSE
    PARSE -->|失败| SPLIT
    SPLIT -->|失败| SINGLE

    FEAT_VAL --> PARSE
    PARSE -->|为空| FALLBACK
    FALLBACK -->|POST 回写| FEAT_VAL
```

### D1 vs KV 分工

| 维度 | D1 | KV |
|---|---|---|
| 模型 | 关系型 SQLite | 键值对 |
| 查询 | SQL（过滤/排序）| 按 key 整取 |
| 一致性 | 强一致 | 最终一致（边缘缓存）|
| GalNavi 用途 | 29 条站点核心数据 | 轮播图 + 站长推荐 |
| 读取延迟 | 略高（需查询）| 极低（边缘缓存）|
| 变更频率 | 中（新增/编辑站点）| 极低（手动调整）|

        D1[("主站D1<br/>站点表")]

    U -->|GET /| W_ENTRY

    U -->|img src| GH

    W_MAIN -->|JSON 响应| U
    W_DETAIL -->|SSR HTML| U

## 七、请求与渲染时序

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant W as 主站 Worker
    participant D1 as D1 数据库
    participant KV as KV 存储

    U->>W: GET /nav/
    W-->>U: SSR HTML 外壳<br/>含内联 CSS + JS

    Note over U: DOMContentLoaded

    par 并行请求
        U->>W: GET /nav/api/nav
        W->>D1: SELECT 站点字段 ORDER BY category
        D1-->>W: 站点数据
        W-->>U: JSON 站点列表
    and
        U->>W: GET /nav/api/hero
        W->>KV: GET 轮播配置
        KV-->>W: 图片 URL 数组
        W-->>U: JSON 轮播图
    and
        U->>W: GET /nav/api/featured
        W->>KV: GET 推荐配置
        KV-->>W: item_key 数组
        W-->>U: JSON 推荐列表
    end

    Note over U: 数据到达后渲染
    U->>U: buildCard 渲染卡片
    U->>U: renderHero 渲染轮播
    U->>U: renderFeatured 渲染推荐
```
