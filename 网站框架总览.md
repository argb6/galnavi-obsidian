## 二、D1 数据库 — 站点核心数据

```mermaid
erDiagram
    SITES {
        TEXT item_key PK "唯一键 (如 joiplay/灵梦御所)"
        TEXT title "标题"
        TEXT category "分类: simulators/websites/tools/company/hanhua"
        TEXT tags "逗号分隔标签 (如 安卓,rpg,多功能)"
        TEXT short_desc "一句话简介"
        TEXT url "主站链接 (模拟器为空)"
        TEXT icon_path "图标 URL"
        TEXT md_content "详情页 Markdown (仅详情页查询)"

    SITES ||--o{ CATEGORY : belongs_to

        TEXT name "分类名"
        TEXT frontend "前端映射 (单数)"
        INT count "条目数"
    }

    TAGS {
        TEXT name "标签名"
        TEXT group "标签分组"
        INT frequency "出现频次"
    }
```

## 三、KV 存储 — 配置型数据

```mermaid
graph LR
    subgraph KV Namespace 1 · 轮播图
        HERO_KEY["key: hero_images"]
        HERO_VAL["value: JSON 数组<br/>[图片URL1, URL2, URL3]"]
    end

    subgraph KV Namespace 2 · 站长推荐
        FEAT_KEY["key: featured_keys"]
        FEAT_VAL["value: JSON 数组<br/>[灵梦御所, touchgal, ...]"]
    end

    subgraph 容错机制
        PARSE["JSON.parse"]
        SPLIT["逗号分割"]
        SINGLE["单元素数组"]
        FALLBACK["标签匹配算法<br/>取含'推荐'标签条目"]
    end

    HERO_VAL --> PARSE
    PARSE -->|失败| SPLIT
    SPLIT -->|失败| SINGLE

    FEAT_VAL --> PARSE
    PARSE -->|为空| FALLBACK
    FALLBACK -->|POST 回写| FEAT_VAL
```

### D1 vs KV 分工

| 维度 | D1 | KV |
|---|---|---|
| 模型 | 关系型 SQLite | 键值对 |
| 查询 | SQL（过滤/排序）| 按 key 整取 |
| 一致性 | 强一致 | 最终一致（边缘缓存）|
| GalNavi 用途 | 29 条站点核心数据 | 轮播图 + 站长推荐 |
| 读取延迟 | 略高（需查询）| 极低（边缘缓存）|
| 变更频率 | 中（新增/编辑站点）| 极低（手动调整）|

```mermaid
flowchart TB
    subgraph 用户端

    end

        W_ENTRY["galnavi.js<br/>/ 入口页"]
        W_MAIN["websearch.js<br/>/nav/ 主站 + API"]
        W_DETAIL["detail.js<br/>/nav/detail/ 详情页"]
        W_HELP["help.js<br/>/nav/help/ 帮助页"]
        W_ABOUT["about.js / moho.js<br/>/about 关于页"]
    end

    subgraph 存储
        D1[("D1 数据库<br/>sites 表<br/>29 条记录<br/>8 个字段")]
        KV_HERO[("KV-轮播图<br/>hero_images<br/>3 张图片 URL")]
        KV_FEAT[("KV-站长推荐<br/>featured_keys<br/>item_key 数组")]
    end

    subgraph 外部资源
        GH["GitHub raw<br/>图标/图片素材"]
    end

    U -->|GET /| W_ENTRY
    U -->|GET /nav/| W_MAIN
    U -->|GET /nav/detail/?item_key=xxx| W_DETAIL
    U -->|GET /nav/help/| W_HELP
    U -->|GET /about| W_ABOUT

    W_MAIN -->|SELECT 7 字段<br/>ORDER BY category| D1
    W_MAIN -->|GET hero_images| KV_HERO
    W_MAIN -->|GET featured_keys| KV_FEAT
    W_DETAIL -->|SELECT title, short_desc,<br/>tags, md_content<br/>WHERE item_key=?| D1
    W_DETAIL -->|POST 写回推荐| KV_FEAT

    U -->|img src| GH

    W_MAIN -->|JSON 响应| U
    W_DETAIL -->|SSR HTML| U
```

## 七、请求与渲染时序

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant W as Worker (websearch.js)
    participant D1 as D1 数据库
    participant KV as KV 存储

    U->>W: GET /nav/
    W-->>U: SSR HTML 外壳 (1.28MB)<br/>含内联 CSS + JS

    Note over U: DOMContentLoaded

    par 并行请求
        U->>W: GET /nav/api/nav
        W->>D1: SELECT 7 字段 ORDER BY category
        D1-->>W: 29 条站点数据
        W-->>U: JSON 站点列表
    and
        U->>W: GET /nav/api/hero
        W->>KV: GET hero_images
        KV-->>W: 图片 URL 数组
        W-->>U: JSON 轮播图
    and
        U->>W: GET /nav/api/featured
        W->>KV: GET featured_keys
        KV-->>W: item_key 数组
        W-->>U: JSON 推荐列表
    end

    Note over U: 数据到达后渲染
    U->>U: buildCard 渲染卡片
    U->>U: renderHero 渲染轮播
    U->>U: renderFeatured 渲染推荐
```
