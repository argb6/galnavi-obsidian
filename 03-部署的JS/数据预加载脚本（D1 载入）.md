# 数据预加载脚本（D1 载入）

> [!info] 位置
> 主站 `/nav/` HTML 的**第 1 行**（`<!DOCTYPE html>` 之前）。这是 GalNavi 最关键的脚本之一——在 HTML 正式解析前就发起 D1 数据请求。

## 职责

1. 定义全局 `escapeHtml` 函数（XSS 防护，详见 [XSS 防护与 escapeHtml](XSS防护与escapeHtml.md)）
2. 异步从 `/nav/api/nav`（D1）获取站点数据
3. 格式化为前端友好的结构
4. 将结果 Promise 挂到 `window.__DATA_PROMISE__`，供主应用 await

## 完整源码

```javascript
<script>
// ==================== HTML 转义函数（防 XSS） ====================
function escapeHtml(str) {
  if (!str) return '';
  return String(str)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}
// ==================== 全新 D1 数据库资料载入与格式化逻辑 ====================
window.__DATA_PROMISE__ = (async function(){
  try {
    // 🌟 精准匹配你的部署路径
    const ctrl1 = new AbortController();
    const t1 = setTimeout(() => ctrl1.abort(), 10000);
    const response = await fetch('/nav/api/nav', { cache: 'no-cache', signal: ctrl1.signal });
    clearTimeout(t1);
    if (!response.ok) throw new Error('HTTP 错误: ' + response.status);
    const d1Data = await response.json();
    if (!Array.isArray(d1Data) || d1Data.length === 0) {
      return null;
    }
    const CAT_MAP = { simulators: 'simulator', websites: 'site', tools: 'tool', company: 'company', hanhua: 'hanhua' };
    const formattedItems = d1Data.map(site => ({
      id: site.item_key,
      cat: CAT_MAP[site.category] || site.category,
      name: site.title,
      desc: site.short_desc || '',
      tags: site.tags ? site.tags.split(',') : [],
      url: site.url || '',
      icon: site.icon_path || ''
    }));return { items: formattedItems };
  } catch (error) {
    return null;
  }
})();
window.__DATA_FP__ = "d1_dynamic_data";
</script>
```

## 逐段解析

### 1. escapeHtml 预定义
放在最前面，因为主应用 JS（行948）也会用到它，且需在渲染卡片前就绪。

### 2. IIFE + async 立即执行
```javascript
window.__DATA_PROMISE__ = (async function(){ ... })();
```
- 用**立即调用的 async 函数表达式**，返回一个 Promise
- 主应用后续 `await window.__DATA_PROMISE__` 即可拿到数据
- 这样**网络请求在 HTML 解析早期就开始**，与 DOM 解析并行

### 3. 超时保护
```javascript
const ctrl1 = new AbortController();
const t1 = setTimeout(() => ctrl1.abort(), 10000);
```
- 10 秒超时，避免 D1 慢时无限等待
- 超时触发 `abort()`，fetch 抛错进入 catch

### 4. 请求配置
```javascript
fetch('/nav/api/nav', { cache: 'no-cache', signal: ctrl1.signal })
```
- `cache: 'no-cache'` — 强制向服务器验证（不直接用浏览器缓存），保证数据新鲜
- 路径 `/nav/api/nav` 是相对路径，命中同源 Worker（详见 [API 端点清单](API端点清单.md)）

### 5. 数据校验
```javascript
if (!Array.isArray(d1Data) || d1Data.length === 0) return null;
```
- 必须是非空数组，否则返回 null
- 主应用据此判断"无数据"状态

### 6. 字段映射（CAT_MAP）
```javascript
const CAT_MAP = { simulators: 'simulator', websites: 'site', tools: 'tool', company: 'company', hanhua: 'hanhua' };
```
将 D1 的**复数 category**（`simulators`/`websites`）映射为前端的**单数 cat**（`simulator`/`site`），与 `data-nav` 视图名一致（见 [路由与页面体系](路由与页面体系.md)）。

| D1 category | 前端 cat | 视图 |
|---|---|---|
| simulators | simulator | 模拟器 |
| websites | site | 站点 |
| tools | tool | 工具 |
| company | company | 会社 |
| hanhua | hanhua | 汉化组 |

### 7. 字段重命名
D1 字段 → 前端字段：
- `item_key` → `id`
- `category` → `cat`（经 CAT_MAP）
- `title` → `name`
- `short_desc` → `desc`
- `tags`（逗号串）→ `tags`（数组，`.split(',')`）
- `url` → `url`
- `icon_path` → `icon`

### 8. 失败处理
```javascript
catch (error) { return null; }
```
- 任何错误（超时/网络/解析）都返回 null，**不抛出**
- 保证主应用 await 时不崩，可走空态

### 9. 指纹标记
```javascript
window.__DATA_FP__ = "d1_dynamic_data";
```
- 一个数据源指纹标记，主应用可据此判断数据来自 D1 动态加载（区别于其他来源）

## 设计亮点

| 设计 | 价值 |
|---|---|
| 放在 DOCTYPE 之前 | 最早发起请求，并行 HTML 解析 |
| 返回 Promise 而非直接 await | 解耦：预加载与消费分离 |
| AbortController 超时 | 防止慢请求拖垮首屏 |
| CAT_MAP 映射 | 数据层与视图层解耦，D1 用复数语义，前端用单数 |
| catch 返回 null | 容错，不阻塞主流程 |
| `cache: 'no-cache'` | 数据时效性优先 |
