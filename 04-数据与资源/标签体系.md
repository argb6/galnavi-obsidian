# 标签体系

> [!info] 概述
> GalNavi 用**自由文本标签**（逗号分隔）描述站点属性，无预定义枚举。标签既是分类补充，也是搜索/过滤的依据。29 条数据共出现约 60+ 种标签。

## 标签的作用

1. **卡片展示**：每张卡片显示前 7 个标签徽章（见 [[03-部署的JS/主应用逻辑脚本（卡片与交互）]] 的 buildCard）
2. **点击筛选**：点标签徽章 → 填入搜索框 → `filterByKeyword` 联动过滤
3. **搜索匹配**：`filterByKeyword` 同时匹配名称与标签
4. **标签云视图**：`buildTagStats` 统计频次，`view-tags` 展示
5. **推荐 fallback**：含"推荐"标签的条目作为站长推荐兜底

## 标签使用特点

1. **自由文本**：无固定枚举，站长按需添加
2. **逗号分隔存储**：`"安卓,rpg,多功能"`，前端 split
3. **大小写敏感**：`ONS` 与 `ons` 视为不同（搜索时 toLowerCase 处理）
4. **中英混用**：`安卓`、`开源`（中文）与 `api`、`MEGA`（英文）并存
5. **粒度不一**：既有宽泛的 `ACG`，也有具体的 `krkr`

## 推荐机制中的标签

```javascript
// featured 的 fallback：取含"推荐"标签的条目
featured = allItems.filter(item => item.tags.includes('推荐')).slice(0, 6);
```
> 注意：当前 29 条数据中**没有"推荐"标签**的条目（featured 走 KV 的 item_keys），所以这个 fallback 实际未触发。但代码预留了用"推荐"标签标记的能力。
