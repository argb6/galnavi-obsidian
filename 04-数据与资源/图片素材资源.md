# 图片素材资源

> [!info] 概述
> GitHub 仓库 `argb6/gal-navigation` 承担了 GalNavi 的**静态资源 CDN** 角色。轮播图、图标、吉祥物、表情包等素材均托管在仓库，通过 `raw.githubusercontent.com` 访问。

## 各素材用途

### 1. hero 轮播图（`docs/hero/`）
- **3 张图片**：hero1.png、hero2.png、hero3.png
- **被 KV 引用**：`/nav/api/hero` 返回的图片 URL 指向：
  ```
  https://raw.githubusercontent.com/argb6/gal-navigation/main/docs/hero/hero1.png
  ```
- **消费方**：[[03-部署的JS/轮播图脚本（Hero Carousel）]] 的 `initHeroCarousel`
- 这是仓库素材与线上 KV 的**直接关联点**

### 2. 项目图标（`assets/`）
- `icon.png` — README 顶部展示、Twitter card 图片
- `logo.png` — 项目主 Logo（454KB，较大）
- `favicon.png` — 网站 favicon 源
- 线上 favicon 实际用内联 base64（见主站 HTML `<link rel="icon" href="data:image/png;base64,...">`）

### 3. 吉祥物（`docs/mascot/`）
- 5 张吉祥物图片（3 png + 2 jpg）
- 推测用于入口页或主站的装饰/品牌形象
- README 的"猫耳摸头杀~~~~~"暗示有猫耳吉祥物

### 4. 表情包（`docs/emoji/`）
- 约 **40 个**表情图片，中文命名（不屑、吃瓜、大哭、喝奶茶、比心、欢呼、害羞、点赞、垂头丧气、发呆、催眠、加油、吐血、喵、嘻嘻、嫉妒、害羞、庆祝、恐惧、投掷、摸头杀、晕厥、欢呼、比心、点赞、生日快乐、生气、画个圈圈诅咒你、疑惑、间笑、白板、睡觉、石化、羡慕、赌气、委屈、震惊、装气、祝贺、祝福等）
- ACG 风格的表情包
- 用途推测：Discord 社区 / 反馈沟通用，或主站某处的彩蛋
- git 历史显示近期在清理这些 emoji（多次 Delete 操作）

### 5. 截图（`docs/*.png`）
- 入口.png、主站.png、详情.png
- 用于 README 展示项目界面
- 是**文档截图**，非线上资源

## GitHub raw 作为 CDN 的机制

```
https://raw.githubusercontent.com/argb6/gal-navigation/main/docs/hero/hero1.png
       └───────────┬───────────┘ └────┬────┘ └─┬─┘ └────┬────┘
            raw 服务域名          仓库 owner  分支   文件路径
```

- GitHub raw 提供**原始文件内容**（无 HTML 包装）
- 走 GitHub CDN，全球可访问
- `main` 分支固定，文件路径稳定
- **免费**，无需额外 CDN 服务

## CSP 与图片源

主站 CSP 的 `img-src`：
```
img-src 'self' data: https:;
```
- `'self'` — 同源图片
- `data:` — base64 内联图片（favicon 用）
- `https:` — **允许任何 https 图片源**

> `img-src https:` 较宽松，因为站点图标来自各站（bing、各站自身），无法预知所有域。这是功能需求与安全性的权衡。详见 [[06-安全与SEO/内容安全策略 CSP]]。

## 站点图标（icon_path）

各站图标 URL 来源多样（见 [[data.json 数据结构]]）：
- 站点自身：`https://www.touchgal.ink/_next/image?...`
- Bing 缩略图：`https://www.bing.com/th/id/ODF...` 或 `https://thfvnext.bing.com/...`
- 第三方图标库：`https://acgnavi.net/site-icons/...`
- Bing 图片搜索：`https://ts4.tc.mm.bing.net/th/id/OIP-C...`

这些**不在仓库**，而是引用外部服务。
