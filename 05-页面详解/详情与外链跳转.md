# 详情与外链跳转

> ！路由
> 
> `GET /nav/detail/?item_key=xxx` — 站点详情页。完全 SSR，独立页面，**不在 sitemap 中**。从主站卡片"介绍详情"按钮进入。

## 页面定位

详情页展示单个站点的**完整信息**，比卡片更详细，是 GalNavi"为每个站点整理信息"承诺的落地。

## 入口

主站卡片的"介绍详情"按钮（详见 [主应用逻辑脚本（卡片与交互）](../03-部署的JS/主应用逻辑脚本（卡片与交互）.md) 的 3. buildCard（item， keyword） —— 卡片 HTML 构建）：
```html
<a href="/nav/detail/?item_key=joiplay" class="btn-detail">介绍详情</a>
```
- 用 `item_key` 作为查询参数
- `encodeURIComponent(item.id)` 编码（支持中文 item_key 如"灵梦御所"）

## 信息丰富度

详情页比卡片多提供：`标签` `名称` `基本信息` `官网链接` `网盘` `插件` `使用说明`

## SSR 特性

- 完全服务端渲染：所有信息直出到 HTML（由 `renderPage()` 生成）
- 数据由 Worker 从 D1 查询 `item_key` 后注入 HTML
- 查询 SQL（参数化绑定，防注入）：
  ```sql
  SELECT title, short_desc, tags, md_content FROM <表名> WHERE item_key = ?
  ```
- `md_content`（Markdown 内容）由 `parseMdContent()` 解析为 branches（分支结构），再渲染为 sections
- 页面含一段业务 JS：对 `.link-entry` 外链绑定**倒计时跳转**

## 外观

- 底色 `--bg: #141e33`（与主站同系）
- 字体：Google Fonts **Noto Sans SC** + **Outfit**（详情页会 `<link>` 拉取；无 CSP meta）
- 卡片 / 面板半透明 + `backdrop-filter`；`.glow` 层若存在则为 `display: none`

## 外链跳转行为（与主站一致的安全设计）

详情页的外链（官网、网盘等，class `link-entry`）**同样有 3 秒倒计时确认层**：

```html
<a class="link-entry" href="..." target="_blank" rel="noopener noreferrer" data-url="...">网盘</a>
```

```javascript
// /nav/detail/：DOMContentLoaded 后对每个 .link-entry 绑定倒计时
document.querySelectorAll('.link-entry[href]').forEach(function(link) {
  link.addEventListener('click', function(e) {
    e.preventDefault();
    // 显示 redirect-overlay，3 秒倒计时
    // 到时 window.open(targetUrl, '_blank', 'noopener,noreferrer')
    // 用户可点"取消跳转"中止
  });
});
```

详见 [外链跳转脚本（Redirect 倒计时）](外链跳转脚本(Redirect倒计时).md)。

> 新版详情页已与主站统一了安全跳转设计，外链均经倒计时确认。这是对早期"详情页直接跳转"的改进。

## 安全性

详情页外链保护：
- `target="_blank"` 新标签打开
- `rel="noopener noreferrer"` 防 opener 劫持 + referer 泄露
- 外链目标由服务端从 D1 取，非用户可控（防注入）

## 未收录 sitemap 的原因

- 详情页数量随收录增长（当前 29 个），全部收录会使 sitemap 臃肿
- 详情页内容可能频繁变化（网盘链接/密码更新）
- 侧重让搜索引擎收录主站分类页，而非每个详情

## 数据来源

服务端处理 `/nav/detail/?item_key=xxx`：
1. 解析 `item_key`（兼容三种格式：`?item_key=xxx`、`?=xxx`、`/detail/xxx` 路径末尾）
2. 从 D1 查询：`SELECT title, short_desc, tags, md_content FROM <表名> WHERE item_key = ?`（参数化绑定）
3. `parseMdContent(md_content)` 将 Markdown 解析为 branches（分支结构）
4. `renderPage()` SSR 注入 HTML 返回
5. 无数据返回 404，缺 item_key 返回 400

> 详情页的网盘/密码/分支等信息来自 D1 的 **`md_content` 字段**（Markdown 内容），该字段**不在**主站 API 返回中（主站只查前 7 个字段），也未公开在仓库 data.json 里。详见 [存储层 D1 与 KV](../02-网站架构/存储层D1与KV.md)。
