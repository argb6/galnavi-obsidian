# 神魔殿堂

> ！路由
>
> `GET /nav/group/` — 独立 SSR 页面（`group.js` Worker），展示神器、魔器、仙器三类特殊资源。使用独立的 D1 数据库绑定。

## 页面定位

神魔殿堂是 GalNavi 的**独立功能页面**，与主站导航系统分离：
- 主题化设计，基于三界（人族、魔族、仙族）的奇幻背景故事
- 展示三类特殊器物：神器、魔器、仙器
- 每件器物可关联多个链接（官网、详情、额外链接 1-3）
- 独立的 D1 数据库绑定（`group` 绑定），与主站站点数据完全分离

## 页面入口

- **主站导航标签**：在 `websearch.js` 中有"神魔"标签链接到 `/nav/group/`
- **Sitemap 收录**：`https://galnavi.top/nav/group/`，priority 0.8
- **SEO 元信息**：
  - `<title>`：`神魔殿堂`
  - `<meta name="description">`：`神器、魔器、仙器总览`

## 三大分类

## 数据结构（group D1 绑定）

### resources 表结构

| 字段 | 类型 | 说明 | 必填 |
|---|---|---|---|
| `id` | INTEGER | 主键 | ✅ |
| `category` | TEXT | 分类（神器/魔器/仙器）| ✅ |
| `name` | TEXT | 器物名称 | ✅ |
| `official_url` | TEXT | 官网链接 | ❌ |
| `details_url` | TEXT | 详情页链接 | ❌ |
| `link1` | TEXT | 额外链接 1 | ❌ |
| `link2` | TEXT | 额外链接 2 | ❌ |
| `link3` | TEXT | 额外链接 3 | ❌ |

### 查询方式

```javascript
const { results } = await env.group.prepare(
  "SELECT * FROM resources ORDER BY category, id"
).all();
```

- **绑定名**：`group`
- **表名**：`resources`
- **排序**：先按 category，再按 id
- **容错**：数据库查询失败时返回空数组

### 2. 动态光晕背景

三个固定的 `glow` 元素，创造奇幻氛围：

```javascript
// 动画：9s 循环，透明度 0.34 ↔ 0.22，缩放 1 ↔ 1.06
.glow.one   // top: -210px, left: -140px, #f59e0b
.glow.two   // right: -170px, bottom: -220px, #ef4444
.glow.three // top: 50%, left: 50%, #8b5cf6
```

### 3. 粒子系统

- 25 个初始粒子，每 200ms 生成一个

- 颜色：`#fbbf24`、`#ef4444`、`#10b981`、`#8b5cf6`、`#3b82f6`
- 动画：从底部飘向顶部，旋转 720°，16s 后移除
- 尺寸：2px - 6px 随机

## 布局优化

### 名称列自动换行
为适配不同长度的器物名称，名称列设置为自适应宽度：

```css
.item-name {
  font-weight: 600;
  color: var(--text);
  width: 1%;
}
```

`width: 1%` 使该列采用最小宽度，内容自动换行，避免过长的名称挤占其他列空间。

### 链接列强制横向排列
为保持链接标签在一行内显示，添加强制横向排列样式：

```css
/* 链接列强制横向排列 */
tbody td:nth-child(5) {
  white-space: nowrap;
}
```

`white-space: nowrap` 确保链接列内的标签不换行，多个 `.link-tag` 在同一行横向排列，保持布局整洁。

## Worker 实现

### 核心逻辑

```javascript
export default {
  async fetch(request, env, ctx) {
    let data = [];
    if (env && env.group) {
      try {
        const { results } = await env.group.prepare(
          "SELECT * FROM resources ORDER BY category, id"
        ).all();
        data = results || [];
      } catch (e) {
        data = [];
      }
    }
    const html = renderPage(data);
    return new Response(html, {
      headers: {
        "Content-Type": "text/html; charset=utf-8",
        "Cache-Control": "public, max-age=3600",
      },
    });
  },
};
```

### 渲染流程

1. 按固定分类顺序遍历：`["神器", "魔器", "仙器"]`
2. 过滤出当前分类的条目
3. 获取分类配置（`CATEGORY_CONFIG`）
4. 检测是否有任何 link 列非空
5. 构建表头（条件包含"链接"列）
6. 构建行（序号、名称、官网、详情、链接）
7. 返回完整 HTML（内联 CSS + JS）

### HTML 转义

```javascript
function escHtml(str) {
  if (!str) return "";
  return str.replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;").replace(/"/g, "&quot;");
}
```

所有数据库值均通过 `escHtml` 转义，防止 XSS。
